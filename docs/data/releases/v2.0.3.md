## 1. Executive Summary for Non-Technical Users

### Main Improvements
This release focuses on **bridge infrastructure improvements** and **cross-chain communication enhancements**:

- **Ethereum Bridge Preparation**: Updated the system to prepare for Ethereum's upcoming "Fulu" network upgrade, ensuring continued compatibility between Polkadot and Ethereum networks
- **Simplified Cross-Chain Operations**: Removed overly restrictive filters that were preventing legitimate cross-chain transactions, making it easier for users and applications to move assets between different blockchains
- **System Maintenance**: Fixed technical issues with weight calculations that improve the accuracy of transaction fee estimates

### Benefits for Users
- **Better Cross-Chain Experience**: Fewer restrictions when moving assets between different blockchain networks
- **Improved Reliability**: More accurate transaction fees and better system performance
- **Future-Proof Bridge**: Continued seamless operation of Ethereum-Polkadot bridge after Ethereum's network upgrade

### Actions Required
- **No immediate action needed** for most users
- The Polkadot Bridge Hub will need to be updated before December 3rd, 2025, but this is handled by network governance

## 2. Key Changes Explained

### Breaking Changes
- **Pool Asset Transactors Removed**: Applications that relied on moving pool assets across chains will no longer be able to do so. This simplifies the system since pool assets weren't designed for cross-chain use anyway.

### Security Improvements
- **Relaxed XCM Transact Filters**: Previously, the system was too restrictive about which cross-chain transactions were allowed, causing legitimate use cases to fail. This has been fixed while maintaining security for critical operations.

### Performance Improvements
- **Fixed Weight Path Calculations**: The system now correctly calculates transaction costs for multi-block election processes, leading to more accurate fee estimates and better resource allocation.

### New Features
- **Ethereum Fulu Fork Support**: The bridge system is now configured to handle Ethereum's upcoming network upgrade, ensuring uninterrupted service.

## 3. Technical Summary for Developers

### Major API/Interface Changes
- **Removed `PoolFungiblesTransactor`**: Pool assets can no longer be transacted via XCM
- **Simplified `AssetTransactors`**: Now only includes `(FungibleTransactor, FungiblesTransactor, ForeignFungiblesTransactor)`
- **Updated Snowbridge Dependencies**: Upgraded to `unstable2507-2` versions with Fulu fork epoch configuration

### Migration Requirements
- **No Runtime Migrations**: This release doesn't require database migrations
- **Dependency Updates**: Projects using pool asset cross-chain functionality need to remove those integrations
- **Weight Path Updates**: Applications using EPMB (Election Provider Multi Block) pallets should verify benchmark paths are correctly resolved

### Technical Improvements
- **Corrected EPMB Weight Paths**: Fixed benchmark paths from `pallet_election_provider_multi_block_*` to `pallet_election_provider_multi_block::*` format
- **Ethereum Client Updates**: Updated beacon primitives and client pallets to support Ethereum's Fulu hard fork
- **Dependency Cleanup**: Removed 199 lines of complex pool asset transactor code, simplifying the codebase
